# Job Radar MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a polished localhost dashboard that stores and displays only remote contract/freelance sales or marketing jobs available to applicants in Sweden, Romania/Bucharest, EMEA/Europe, or worldwide.

**Architecture:** A single Next.js application owns the UI and server-side sync pipeline. Replaceable connectors normalize public job data, pure rules classify strict matches, and a small SQLite repository persists accepted jobs and sync summaries.

**Tech Stack:** Next.js 16.2, React 19, TypeScript 5, `better-sqlite3`, Vitest, ESLint, plain CSS

---

## File map

- `app/layout.tsx`: root metadata and document shell.
- `app/page.tsx`: server-side dashboard query and page composition.
- `app/actions.ts`: manual sync server action with user-safe errors.
- `app/api/cron/sync/route.ts`: protected scheduled-sync endpoint.
- `app/components/dashboard.tsx`: client-side search, selectors, sync feedback, and job cards.
- `app/globals.css`: all visual tokens, responsive layout, and interaction states.
- `lib/job-radar/types.ts`: shared source, matched-job, connector, and sync types.
- `lib/job-radar/matcher.ts`: strict explainable match rules and canonical URL helper.
- `lib/job-radar/matcher.test.ts`: matcher and URL-normalization coverage.
- `lib/job-radar/fetch.ts`: timeout-aware JSON fetch and redirect resolution.
- `lib/job-radar/connectors/*.ts`: one adapter per source plus enabled-source configuration.
- `lib/job-radar/connectors/connectors.test.ts`: fixture-based source mapping tests.
- `lib/job-radar/db.ts`: SQLite initialization, queries, and upserts.
- `lib/job-radar/sync.ts`: connector orchestration, partial failures, matching, and persistence.
- `lib/job-radar/sync.test.ts`: orchestration tests using injected connectors and repository.
- `lib/job-radar/schedule.ts`: Stockholm-time next-run calculation.
- `lib/job-radar/schedule.test.ts`: summer/winter and boundary-time tests.
- `scripts/sync.ts`: one-shot CLI entry point.
- `scripts/scheduler.ts`: long-lived twice-daily scheduler.
- `README.md`: setup, configuration, commands, and scheduling behavior.
- `QA.md`: the required short localhost QA script.

- [ ] **Step 3: Implement the connectors**

Every request uses a 15-second timeout. Arbeitnow `/apply` and Jooble tracking links are resolved only after a job passes matching; the final URL is canonicalized before storage.
